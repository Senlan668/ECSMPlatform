# AI Slice — Windows 最小启动流程

> 前提:已装好本地 PostgreSQL、Python 3.11+、uv、Node.js 18+、FFmpeg,以及 DeepSeek / Groq 的 API Key。

## 0. FFmpeg 准备(Windows 关键)

后端切片需要调用 `ffmpeg.exe` 和 `ffprobe.exe`。Windows 没有官方安装包,需要手动下载预编译版本。

### 下载

打开官方下载页:<https://ffmpeg.org/download.html#build-windows>

页面里 **Windows EXE Files** 下有两个可选源:

- **Windows builds from gyan.dev** (推荐,更新快、版本全)
- **Windows builds by BtbN**

任选其一,下载 **release-full** 或 **full_build** 的 7z/zip 压缩包(包含 `ffmpeg.exe` / `ffprobe.exe` / `ffplay.exe`)。

### 解压 + 配置

解压到任意目录,例如:

```
D:\ffmpeg\ffmpeg-2025-12-22-git-c50e5c7778-full_build\bin\
  ├─ ffmpeg.exe
  ├─ ffprobe.exe
  └─ ffplay.exe
```

把 **`bin` 目录的绝对路径**写到 `backend\.env` 的 `FFMPEG_BIN_DIR`,路径用正斜杠 `/`:

```dotenv
FFMPEG_BIN_DIR=D:/ffmpeg/ffmpeg-2025-12-22-git-c50e5c7778-full_build/bin
```

### 验证

```powershell
& "D:\ffmpeg\ffmpeg-2025-12-22-git-c50e5c7778-full_build\bin\ffmpeg.exe" -version
```

能打印出版本号就 OK。

> 💡 也可以把 `bin` 目录加到系统 PATH,这样 `FFMPEG_BIN_DIR` 可以留空,后端会按 PATH 兜底查找。

## 1. 数据库准备(只需一次)

直接用本地 PostgreSQL 的 `postgres` 超级用户建库即可:

```powershell
$env:PGPASSWORD='你的postgres密码'
psql -h localhost -U postgres -c "CREATE DATABASE ai_slice;"
```

对应的 `backend\.env` 里:

```dotenv
DATABASE_URL=postgresql+asyncpg://postgres:你的postgres密码@localhost:5432/ai_slice
```

> 数据库已存在会报 `database "ai_slice" already exists`,忽略即可。

## 2. 后端

```powershell
cd backend
Copy-Item .env.example .env
# 编辑 .env 填入 DEEPSEEK_API_KEY / GROQ_API_KEY / FFMPEG_BIN_DIR
uv sync
uv run alembic upgrade head
uv run uvicorn app.main:app --reload --port 8001
```

## 3. 前端(新开一个 PowerShell 窗口)

```powershell
cd frontend
npm install
npm run dev
```

打开 <http://localhost:5173>。
