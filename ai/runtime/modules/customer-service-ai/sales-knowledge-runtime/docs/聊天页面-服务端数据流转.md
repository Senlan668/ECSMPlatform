# /chat 聊天页面 — 服务端数据流转

## 一、总览

```
微信 SQLite 数据库 ──ETL导入──▶ PostgreSQL ──API查询──▶ JSON 响应 ──▶ 前端页面
   (源头)                         (存储)                (输出)
```

涉及 **3 张核心表**：`contacts`（联系人）、`sessions`（会话）、`raw_chats`（原始消息）。

---

## 二、PostgreSQL 表结构

### 2.1 `contacts` — 联系人表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer (PK) | 自增主键 |
| wxid | String(100) | 微信ID，唯一索引 |
| alias | String(100) | 微信号 |
| nickname | String(200) | 昵称 |
| remark | String(200) | 备注名 |
| display_name | String(200) | 显示名称（优先级：备注 > 昵称） |
| avatar_url | Text | 头像URL |
| is_chatroom | Boolean | 是否群聊 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 2.2 `sessions` — 会话表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer (PK) | 自增主键 |
| session_id | String(100) | 会话ID（wxid 或 群ID），唯一索引 |
| display_name | String(200) | 显示名称 |
| is_chatroom | Boolean | 是否群聊 |
| last_message | Text | 最后一条消息（截取前100字） |
| last_time | BigInteger | 最后消息的 Unix 时间戳（秒） |
| message_count | Integer | 消息总数 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 2.3 `raw_chats` — 原始消息表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer (PK) | 自增主键 |
| local_id | Integer | 微信本地消息ID |
| session_id | String(100) | 会话ID，索引 |
| sender_wxid | String(100) | 发送者wxid（自己发的为 `self`） |
| sender_name | String(200) | 发送者名称（自己发的为 `我`） |
| content | Text | 消息内容（原始内容） |
| msg_type | Integer | 消息类型，见下方枚举 |
| is_sender | Boolean | 是否是自己发送 |
| timestamp | BigInteger | Unix 时间戳（秒），索引 |
| display_content | Text | 显示内容（如引用消息） |
| extra_data | JSONB | 额外数据 |
| source_db | String(20) | 来源数据库（MSG0~MSG5） |
| msg_server_id | BigInteger | 微信服务器消息ID，用于关联语音文件 |
| voice_path | String(300) | 语音文件相对路径 |
| status | String(20) | 审核状态：pending/approved/rejected/edited |
| clean_content | Text | 清洗后的内容（人工修改） |
| auto_category | String(50) | 自动分类 |
| auto_flags | JSONB | 自动标记 |
| reviewed_by | String(100) | 审核人 |
| reviewed_at | DateTime | 审核时间 |
| created_at | DateTime | 创建时间 |

**复合索引**：`(session_id, timestamp)`、`(status, session_id)`

**msg_type 消息类型枚举**：

| 值 | 含义 |
|----|------|
| 1 | 文本 |
| 3 | 图片 |
| 34 | 语音 |
| 43 | 视频 |
| 47 | 表情 |
| 49 | 链接/文件 |

## 三、数据来源 — ETL 导入流程

数据源是微信本地 SQLite 数据库，通过 `WeChatETL` 类（`app/services/etl.py`）完成导入。

### 3.1 ETL 执行步骤

```
Step 0   ─ （可选）清除旧数据：DELETE raw_chats / sessions / contacts
Step 0.5 ─ 扫描语音目录，加载 .mp3 文件名集合
Step 1   ─ 读 MicroMsg.db → 写入 contacts 表
Step 2   ─ 读 ChatRoomUser.db → 内存缓存群成员映射
Step 3   ─ 遍历 MSG0~MSG5.db → 写入 raw_chats 表
Step 4   ─ 聚合 raw_chats → 写入 sessions 表
Step 5   ─ 语音补链：为 voice_path 为空的语音消息匹配文件
```

### 3.2 关键转换逻辑

#### 联系人导入（MicroMsg.db → contacts）
```
SQLite: Contact 表 → (UserName, Alias, NickName, Remark, SmallHeadImgUrl)
                           ↓
转换: display_name = remark || nickname || alias || wxid
      is_chatroom  = wxid.endswith('@chatroom')
                           ↓
PostgreSQL: contacts 表 (UPSERT by wxid)
```

#### 消息导入（MSG0~5.db → raw_chats）
```
SQLite: MSG 表 → (localId, StrTalker, StrContent, CreateTime, Type, 
                   SubType, IsSender, DisplayContent, BytesExtra, MsgSvrID)
                           ↓
过滤: timestamp >= 1759248000（仅保留 2025年10月之后）
                           ↓
转换:
  - session_id = StrTalker（对话方的wxid）
  - 群消息解析: 从 content 中拆分 "wxid:\n正文" 得到发送者
  - 自己发的: sender_wxid='self', sender_name='我'
  - 私聊对方: sender_wxid=talker, sender_name=通讯录查找
  - 语音匹配: msg_type==34 时，用 MsgSvrID 在语音目录中查找 .mp3
                           ↓
去重: source_db + local_id 联合去重
                           ↓
PostgreSQL: raw_chats 表 (批量写入, 1000条/批)
```

#### 会话构建（raw_chats 聚合 → sessions）
```
SELECT session_id, COUNT(*), MAX(timestamp) FROM raw_chats GROUP BY session_id
                           ↓
补充:
  - display_name = 通讯录中的名称
  - is_chatroom = session_id.endswith('@chatroom')
  - last_message = 最后一条消息内容[:100]
                           ↓
PostgreSQL: sessions 表 (UPSERT by session_id)
```

## 四、API 接口 — 数据查询与序列化

路由文件：`app/routers/chats.py`（前缀 `/api/chats`）和 `app/routers/search.py`（前缀 `/api/search`）。

### 4.1 获取会话列表

**`GET /api/chats/sessions`**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| page | 1 | 页码 |
| page_size | 50 | 每页条数（上限100） |
| search | 无 | 按 display_name 模糊搜索 |
| session_type | 无 | all / private / chatroom |
| exclude_chatroom | true | 默认排除群聊 |

**查询逻辑**：
```sql
SELECT * FROM sessions
WHERE last_time >= 1759248000          -- 仅2025年10月后
  AND is_chatroom = false              -- 默认排除群聊
  AND display_name ILIKE '%search%'    -- 可选搜索
ORDER BY last_time DESC
LIMIT page_size OFFSET (page-1)*page_size
```

**返回 JSON 结构**：
```json
{
  "items": [
    {
      "id": 1,
      "session_id": "wxid_xxx",
      "display_name": "张三",
      "is_chatroom": false,
      "last_message": "好的，明天见",
      "last_time": 1760000000,
      "message_count": 128
    }
  ],
  "total": 42,
  "page": 1,
  "page_size": 50,
  "has_more": false
}
```

### 4.2 获取聊天记录

**`GET /api/chats/history/{session_id}`**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| page | 1 | 页码 |
| page_size | 50 | 每页条数（上限200） |
| before_time | 无 | 早于此时间戳 |
| after_time | 无 | 晚于此时间戳 |
| msg_type | 无 | 消息类型筛选 |

**查询逻辑**：
```sql
SELECT * FROM raw_chats
WHERE session_id = :session_id
  AND timestamp < :before_time    -- 可选
  AND timestamp > :after_time     -- 可选
  AND msg_type = :msg_type        -- 可选
ORDER BY timestamp DESC
LIMIT page_size OFFSET (page-1)*page_size
```

> 查询后做了一次 `reverse()`，使返回结果按时间正序（旧→新），方便前端渲染聊天气泡。

**返回 JSON 结构**：
```json
{
  "items": [
    {
      "id": 100,
      "local_id": 5678,
      "session_id": "wxid_xxx",
      "sender_wxid": "self",
      "sender_name": "我",
      "content": "你好，课程还有优惠吗？",
      "msg_type": 1,
      "is_sender": true,
      "timestamp": 1760000000,
      "display_content": null,
      "voice_path": null
    }
  ],
  "total": 128,
  "page": 1,
  "page_size": 50,
  "has_more": true
}
```

### 4.3 获取消息上下文（搜索跳转用）

**`GET /api/chats/context/{message_id}`**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| context_size | 25 | 目标消息前后各取多少条 |

**查询逻辑**：
```
1. 根据 message_id 找到目标消息
2. 查 timestamp <= 目标 的前 context_size+1 条（含自身）
3. 查 timestamp > 目标 的后 context_size 条
4. 合并 + 去重 + 按 (timestamp, id) 排序
```

**返回 JSON 结构**：
```json
{
  "target_message_id": 100,
  "session_id": "wxid_xxx",
  "items": [ /* MessageResponse 数组 */ ],
  "total": 50
}
```

### 4.4 全文搜索消息

**`GET /api/search/messages`**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| q | 必填 | 搜索关键词 |
| page | 1 | 页码 |
| page_size | 20 | 每页条数 |
| session_id | 无 | 限定会话 |
| start_time / end_time | 无 | 时间范围 |
| exclude_chatroom | true | 排除群聊 |

**查询逻辑**：
```sql
SELECT * FROM raw_chats
WHERE content ILIKE '%关键词%'
  AND session_id NOT LIKE '%@chatroom'  -- 默认排除群聊
ORDER BY timestamp DESC
```

**额外处理**：
- 对每条结果关联查询 `sessions` 表拿 `display_name` 作为 `session_name`
- 生成 `highlight` 字段：截取关键词前后各30字符 + 省略号

**返回 JSON 结构**：
```json
{
  "items": [
    {
      "id": 100,
      "session_id": "wxid_xxx",
      "session_name": "张三",
      "sender_name": "我",
      "content": "你好，课程还有优惠吗？...",
      "timestamp": 1760000000,
      "msg_type": 1,
      "highlight": "...课程还有<b>优惠</b>吗？..."
    }
  ],
  "total": 5,
  "query": "优惠",
  "page": 1,
  "page_size": 20,
  "has_more": false
}
```

### 4.5 语音文件服务

**`GET /api/chats/voice/{filename}`**

直接返回 `.mp3` 文件流，前端用 `<audio>` 播放。路径安全校验：只允许 `.mp3` 后缀，禁止路径遍历。

### 4.6 其他辅助接口

| 接口 | 说明 |
|------|------|
| `GET /api/chats/sessions/{session_id}` | 获取单个会话详情 |
| `GET /api/chats/contacts` | 联系人列表（分页，可搜索） |
| `GET /api/chats/contacts/{wxid}` | 单个联系人详情 |
| `GET /api/search/sessions` | 搜索会话/联系人名称 |

## 五、数据流转全景图

```
┌─────────────────────────────────────────────────────────────────┐
│                     微信本地 SQLite 文件                         │
│  MicroMsg.db (通讯录)  │  MSG0~5.db (消息)  │  Voice/*.mp3 (语音) │
└───────────┬────────────┴────────┬───────────┴────────┬──────────┘
            │                     │                     │
            ▼                     ▼                     ▼
┌───────────────────── ETL (app/services/etl.py) ────────────────┐
│                                                                 │
│  1. MicroMsg.Contact                                            │
│     → display_name = remark || nickname || alias                │
│     → UPSERT into contacts                                     │
│                                                                 │
│  2. MSG.MSG 表 (×6个库)                                          │
│     → 过滤 timestamp < 2025-10-01                                │
│     → 群消息拆分: "wxid:\n正文" → sender_wxid + clean_content    │
│     → 语音匹配: msg_server_id → Voice/{id}.mp3                  │
│     → 去重(source_db+local_id) → INSERT into raw_chats          │
│                                                                 │
│  3. 聚合 raw_chats                                               │
│     → GROUP BY session_id → UPSERT into sessions                │
│                                                                 │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
┌─────────────────── PostgreSQL 数据库 ──────────────────────────┐
│                                                                 │
│  contacts 表 ◄──────── sessions 表 ◄──────── raw_chats 表       │
│  (联系人信息)    wxid    (会话摘要)   session_id  (全量消息)       │
│                                                                 │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
┌─────────── FastAPI 路由 (app/routers/chats.py) ────────────────┐
│                                                                 │
│  GET /sessions          sessions 表分页查询                      │
│         ↓ 序列化: SQLAlchemy ORM → SessionResponse (Pydantic)    │
│                                                                 │
│  GET /history/{sid}     raw_chats 按 session_id 分页查询          │
│         ↓ 序列化: ORM → MessageResponse，然后 reverse() 正序输出   │
│                                                                 │
│  GET /context/{mid}     raw_chats 按 timestamp 前后取 N 条        │
│         ↓ 序列化: 合并去重排序 → MessageResponse 列表              │
│                                                                 │
│  GET /search/messages   raw_chats ILIKE 全文搜索                  │
│         ↓ 关联 sessions 拿名称，截取高亮片段                       │
│                                                                 │
└────────────────────────────┬────────────────────────────────────┘
                             ▼
                     JSON Response → 前端
```

## 六、ORM → JSON 序列化规则

没有复杂的转换层。使用 Pydantic `model_validate()` 直接从 SQLAlchemy ORM 对象映射：

| ORM 模型 | Pydantic 响应模型 | 差异说明 |
|----------|-------------------|----------|
| Session | SessionResponse | 1:1 映射，无额外处理 |
| RawChat | MessageResponse | 裁剪字段：只暴露 13 个字段，隐藏 status/clean_content/auto_flags 等后台字段 |
| Contact | ContactResponse | 1:1 映射，隐藏时间戳字段 |
| RawChat | SearchResultItem | 额外关联 sessions 拿 session_name，额外计算 highlight 字段 |

所有列表接口统一使用 `PaginatedResponse` 包装：
```python
{
    "items": [...],     # 数据列表
    "total": int,       # 总数
    "page": int,        # 当前页
    "page_size": int,   # 每页条数
    "has_more": bool    # 是否还有更多
}
```
