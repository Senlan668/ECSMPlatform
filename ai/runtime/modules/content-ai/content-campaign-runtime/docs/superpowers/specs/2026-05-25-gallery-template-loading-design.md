# Gallery And Template Loading Design

## Goal

降低服务器首次打开作品库和左侧“模板中心”的可感知等待时间，同时保留详情页原图清晰度和现有模板操作能力。

## Confirmed Bottlenecks

- 作品列表接口已返回 `thumbnail_url`，但 `GalleryCard` 使用 `image_url` 渲染网格图片，列表直接下载原图。
- 模板中心初始标签是“我的模板”，但页面调用 `scope=all`，在个人模板出现前等待公共模板一起返回。
- 旧作品可能没有缩略图字段；仅改前端会让这些记录继续下载原图。

## Design

### Gallery Grid

- 列表卡片使用 `thumbnail_url || image_url` 作为展示源，并为列表图片启用浏览器懒加载和异步解码。
- 详情弹窗继续使用 `image_url`，不牺牲查看与下载质量。
- 新作品继续沿用保存时生成缩略图的现有逻辑。

### Template Center

- 页面初始只请求 `scope=mine`，尽快显示当前用户模板。
- 用户第一次切换到“公共模板”时才请求 `scope=system`。
- 在页面生命周期内分别保存两类模板结果；重复切换标签不重复请求。
- 新建、编辑、删除、复制、发布只更新对应列表；发布后切换公共模板时仍确保公共数据已加载。
- 模板封面启用懒加载和异步解码。

### Existing Gallery Data

- 在 `GalleryService` 增加可重复执行的缩略图补齐方法，仅处理缺少 `thumbnail_url` 且存在原图的记录。
- 提供命令行脚本供服务器发布后手动执行。脚本分批更新记录，不把历史处理放入页面请求链路。

## Error Handling

- 缩略图生成失败时保留原图回退路径，并统计跳过数量。
- 某一模板标签加载失败只影响该标签；已加载的数据继续显示，后续切换可重新请求。

## Verification

- 前端测试确认作品卡片引用缩略图、模板中心使用分 scope 加载与缓存。
- 后端测试确认缩略图补齐只更新缺失记录并提交可用缩略图地址。
- 运行前端生产构建和相关 Python 测试。
