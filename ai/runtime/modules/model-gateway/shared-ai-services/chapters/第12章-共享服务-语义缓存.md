# 第 12 章：共享服务五 — 语义缓存

> 一句话：用向量相似度匹配"问过的类似问题"，直接返回缓存结果，省掉重复的 LLM 调用，大幅降低成本。

---

## 12.1 语义缓存 vs 传统缓存

```
传统缓存（精确匹配）：
  "北京今天天气怎么样" → 命中
  "北京今天天气如何"   → 未命中 ❌（差一个字就不行）

语义缓存（相似度匹配）：
  "北京今天天气怎么样" → 命中
  "北京今天天气如何"   → 命中 ✅（语义相同）
  "今天北京天气咋样"   → 命中 ✅（语义相同）
  "上海今天天气怎么样" → 未命中 ✅（语义不同，正确）
```

---

## 12.2 工作原理

```
用户查询
  │
  ▼
① 向量化查询文本
  │
  ▼
② 在缓存向量库中搜索相似查询
  │
  ├── 相似度 ≥ 阈值 → ③ 返回缓存结果（不调用 LLM）
  │
  └── 相似度 < 阈值 → ④ 调用 LLM → ⑤ 结果存入缓存 → 返回
```

**核心参数**：相似度阈值

```
阈值高（0.98）：几乎只有完全相同的问题才命中
  → 命中率低，但质量高，不会返回错误答案

阈值低（0.85）：稍有不同也能命中
  → 命中率高，但可能返回不够精确的缓存答案

推荐起步值：0.92 ~ 0.95
根据实际效果调优
```

## 12.3 架构设计

```
                LLM 网关
                   │
                   ▼
            语义缓存层（拦截器）
           ┌───────┴───────┐
           │               │
        缓存命中         缓存未命中
           │               │
           ▼               ▼
        返回缓存        调用 LLM
                           │
                           ▼
                      结果写入缓存
                           │
                           ▼
                        返回结果
```

语义缓存通常作为 **LLM 网关的中间件** 存在，而不是独立的 MCP Server。它拦截 `chat_completion` 请求，在调用 LLM 之前先查缓存。

数据存储：

```
缓存条目 = {
  query_vector: [0.12, -0.34, ...],   // 查询的向量
  query_text: "北京今天天气怎么样",     // 原始查询（用于日志）
  response: "北京今天晴，22°C...",      // LLM 的回答
  model: "gpt-4o",                     // 使用的模型
  project_id: "proj_A",               // 来源项目
  created_at: "2025-01-15T10:00:00",  // 创建时间
  ttl: 3600                            // 过期时间（秒）
}
```

---

## 12.4 缓存失效策略

| 策略 | 说明 |
|---|---|
| **TTL 过期** | 缓存条目设置生存时间（如 1 小时），到期自动失效 |
| **数据变更** | 当底层数据更新时主动清除相关缓存 |
| **版本隔离** | 切换模型或 Prompt 版本后，旧缓存自动失效 |
| **手动清除** | 管理员主动清除特定缓存 |

**不同场景的 TTL 参考**：

```
天气查询：     30 分钟（数据频繁变化）
知识库问答：   24 小时（知识库不频繁更新）
通用常识：     7 天（几乎不变）
实时数据查询： 不缓存
```

## 12.5 成本效果测算

```
假设：
  每天 10,000 次 LLM 调用
  平均每次 $0.01（GPT-4o，约 1000 tokens）
  语义缓存命中率 35%

节省：
  10,000 × 35% × $0.01 = $35/天
  每月节省：$1,050
  每年节省：$12,600

额外成本：
  向量数据库：~$50/月
  Embedding 调用：~$20/月

净节省：~$980/月
```

命中率随时间增长 —— 累积的缓存越多，命中率越高。典型项目 3 个月后命中率可达 40-60%。

> **适用判断**：如果你的场景中有大量重复或相似查询（客服、FAQ、知识问答），语义缓存效果显著。如果每个查询都高度独特（创意写作），效果有限。
