# 第 14 章：认证、鉴权与安全

> 一句话：统一管理 API Key，按项目/用户分级授权，对敏感数据脱敏，所有操作留审计日志。

---

## 14.1 API Key 统一管理

```
之前：Key 散落各处
  项目 A 的 .env → OPENAI_KEY=sk-xxx
  项目 B 的服务器 → OPENAI_KEY=sk-yyy
  开发者笔记本 → OPENAI_KEY=sk-zzz

之后：Key 集中在网关
  ┌──────────────┐
  │   密钥保险箱   │
  │              │
  │ OpenAI Key ×3│ ← 网关持有，自动轮换
  │ Claude Key ×2│
  │ Gemini Key ×1│
  └──────────────┘
       │
  项目只需要一个"项目令牌"来访问网关
  项目不接触任何 LLM 厂商的 Key
```

好处：
- Key 泄露范围缩小到网关一处
- Key 轮换只改网关，项目无感知
- 可以按项目追踪 Key 使用量

---

## 14.2 分级访问控制

```
三级权限模型：

  项目级（Project）
    │ 这个项目能访问哪些 MCP Server？
    │ 项目 A → LLM ✅ RAG ✅ 记忆 ✅ 文件 ❌
    │ 项目 B → LLM ✅ RAG ✅ 记忆 ❌ 文件 ✅
    │
    ├── Tool 级（Tool）
    │   │ 这个项目能调用哪些 Tool？
    │   │ 项目 A → query_database ✅ delete_record ❌
    │   │
    │   └── 操作级（Action）
    │       │ 具体操作的限制
    │       │ query_database → 只允许 SELECT（禁止 DELETE）
    │       │ send_email → 每天最多 100 封
```

## 14.3 敏感数据脱敏

LLM 调用链上可能经过敏感数据，需要在网关层拦截：

```
用户输入："帮我查一下手机号 13812345678 的订单"
                    │
                    ▼
              网关 PII 过滤器
                    │
                    ▼
  发送给 LLM："帮我查一下手机号 [PHONE_1] 的订单"
                    │
                    ▼
  LLM 返回："[PHONE_1] 的最近订单是…"
                    │
                    ▼
              网关还原 PII
                    │
                    ▼
  返回用户："13812345678 的最近订单是…"
```

需要过滤的 PII 类型：

| 类型 | 示例 | 处理 |
|---|---|---|
| 手机号 | 13812345678 | 替换为 [PHONE] |
| 身份证 | 110101199001011234 | 替换为 [ID_CARD] |
| 银行卡 | 6222021234567890 | 替换为 [BANK_CARD] |
| 邮箱 | user@example.com | 替换为 [EMAIL] |
| 地址 | 北京市朝阳区xx路xx号 | 视策略决定是否过滤 |

## 14.4 审计日志

每一次 MCP 调用都要记录审计日志：

```json
{
  "trace_id": "abc-123",
  "timestamp": "2025-01-15T10:30:00Z",
  "project_id": "proj_A",
  "user_id": "user_456",
  "action": "tools/call",
  "tool_name": "query_database",
  "input": { "sql": "SELECT * FROM orders WHERE ..." },
  "output_summary": "返回 42 条记录",
  "model_used": "gpt-4o",
  "tokens": { "input": 150, "output": 300 },
  "latency_ms": 1200,
  "status": "success",
  "pii_detected": ["PHONE"],
  "risk_level": "low"
}
```

审计日志的用途：
- 安全审查：谁在什么时候做了什么
- 合规要求：金融/医疗等行业的监管需求
- 故障排查：出了问题可以回溯完整链路
- 成本归属：按项目/用户统计消耗
