# 第 20 章：平台化演进 — 从共享服务到 AI 中台

> 一句话：共享 MCP 服务是起点，终局是一个可视化编排、自助接入、有生态的 AI 开发平台。

---

## 20.1 演进路径

```
阶段 1：共享服务（本教程覆盖的）
  几个核心 MCP Server 共享
  配置文件接入
  命令行管理
  │
  ▼
阶段 2：服务平台
  Web 管理控制台
  自助接入流程
  可视化监控看板
  │
  ▼
阶段 3：开发平台
  可视化工作流编排
  低代码搭建 AI 应用
  应用市场和生态
  │
  ▼
阶段 4：AI 中台
  对外输出 AI 能力
  多租户 SaaS 化
  完整的开发者生态
```

> **务实建议**：大多数团队在阶段 1-2 就够用了。不要为了"中台"而中台。

## 20.2 可视化工作流编排

```
从代码编排：
  result = await rag_search(query)
  context = format_context(result)
  answer = await llm_chat(context + query)
  await save_memory(user_id, answer)
  await send_notification(user_id, answer)

到可视化编排：
  ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
  │ RAG  │───►│ 格式化│───►│ LLM  │───►│ 保存 │
  │ 检索 │    │ 上下文│    │ 对话 │    │ 记忆 │
  └──────┘    └──────┘    └──────┘    └──┬───┘
                                         │
                                    ┌────┴────┐
                                    │ 发通知   │
                                    └─────────┘

每个节点 = 一个 MCP Tool 调用
连线 = 数据流向
非开发者也能搭建 AI 工作流
```

技术实现：本质上是一个 DAG（有向无环图）执行引擎，节点是 MCP Tool 调用。

## 20.4 开发者门户

```
开发者自助流程：

  ① 注册项目 → 获取 project_id 和 API Token
  ② 浏览市场 → 选择需要的 Tool 和知识库
  ③ 一键接入 → 配置权限和配额
  ④ 集成开发 → 按文档接入 MCP Client
  ⑤ 测试调试 → 在线 Playground 测试
  ⑥ 上线监控 → 看板查看调用量和质量

开发者门户提供：
  📖 API 文档（自动从 MCP Server 生成）
  🎮 在线 Playground（即时测试 Tool 调用）
  📊 用量看板（实时查看消耗和质量）
  🔔 告警配置（自定义告警规则）
  📦 SDK 下载（Python / TypeScript / Go）
```

## 20.5 对外输出 AI 能力

当内部平台成熟后，可以考虑对外输出：

```
内部使用                          对外输出
─────────                        ─────────
内部项目接入                      外部客户/合作伙伴接入
内部 Token 计量                   商业化计费
内部权限控制                      租户完全隔离
内部监控                          SLA 保障

从"共享服务"到"AI 能力中台"到"AI 服务商"
```

> 这是一个长期愿景。大多数团队不需要走到这一步。**先把内部的共享服务做好，已经能带来巨大价值。**

## 全书回顾

```
第一部分（1-3）：MCP 是什么？怎么用？
  → 建立基础认知

第二部分（4-5）：单项目 MCP 长什么样？有什么痛点？
  → 理解问题

第三部分（6-7）：如何识别通用服务？整体架构怎么设计？
  → 架构思维

第四部分（8-13）：六大共享服务的设计
  → LLM 网关、RAG、记忆、Prompt、缓存、Tool 注册

第五部分（14-17）：治理和运维
  → 安全、日志、成本、评估

第六部分（18-20）：高可用、性能、平台化
  → 进阶演进
```

**核心思想**：MCP 的价值不只是让 LLM 调用工具，更在于通过标准协议将 AI 能力抽象为**可共享、可治理、可演进**的基础设施。
