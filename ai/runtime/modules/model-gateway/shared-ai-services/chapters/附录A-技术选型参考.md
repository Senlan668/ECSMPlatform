# 附录 A：技术选型参考

---

## 向量数据库对比

| | Milvus | Qdrant | Weaviate | pgvector | Chroma |
|---|---|---|---|---|---|
| **语言** | Go/C++ | Rust | Go | C (PG插件) | Python |
| **部署复杂度** | 高（依赖 etcd/MinIO） | 低（单二进制） | 中 | 最低（PG插件） | 最低（嵌入式） |
| **百万级性能** | ★★★★★ | ★★★★★ | ★★★★☆ | ★★★☆☆ | ★★☆☆☆ |
| **混合检索** | ✅ | ✅ | ✅ | ⚠️ 需拼接 | ❌ |
| **多租户** | ✅ partition | ✅ collection | ✅ tenant | ⚠️ 手动 | ❌ |
| **适合场景** | 大规模生产 | 中等规模首选 | 多模态 | 数据量小，不想加组件 | 原型/开发 |

**推荐**：
- 数据量 < 100 万且已有 PG → **pgvector**
- 中等规模，追求简单 → **Qdrant**
- 大规模生产 → **Milvus**

## MCP SDK 对比

| | Python SDK | TypeScript SDK |
|---|---|---|
| **包名** | `mcp` | `@modelcontextprotocol/sdk` |
| **高级 API** | `FastMCP`（装饰器风格，极简） | `McpServer`（类型安全，Zod 校验） |
| **低级 API** | `Server` | `Server` |
| **代码量** | 约为 TS 的 60% | 更多但类型更严格 |
| **生态** | 数据/AI 项目 | Web/全栈项目 |
| **调试工具** | `mcp dev`（内置） | Inspector（需 npx） |

**推荐**：快速原型用 Python，生产级 Web 服务用 TypeScript。

## LLM 网关开源方案对比

| | LiteLLM | OneAPI | 自研 |
|---|---|---|---|
| **语言** | Python | Go | - |
| **模型支持** | 100+ 模型 | 主流模型 | 按需 |
| **接口格式** | OpenAI 兼容 | OpenAI 兼容 | 自定义 |
| **Web 管理** | 有（Proxy 模式） | 有（功能丰富） | 需开发 |
| **Key 管理** | ✅ | ✅ | 需开发 |
| **负载均衡** | ✅ | ✅ | 需开发 |
| **中文社区** | 一般 | 活跃 | - |
| **适合** | 快速集成 | 国内团队 | 有特殊需求 |

**推荐**：
- 国内团队 → **OneAPI**（中文文档好，社区活跃）
- 国际化团队 → **LiteLLM**（模型覆盖最广）
- 在上面包一层 MCP Server 接口即可

## 消息队列选型

| | Redis Streams | RabbitMQ | Kafka |
|---|---|---|---|
| **适合** | 轻量异步 | 通用消息 | 大吞吐日志 |
| **运维** | 最低（已有 Redis） | 中 | 高 |
| **推荐场景** | 异步日志、通知 | Tool 调用解耦 | 日志管道 |

**推荐**：初期用 Redis Streams（不引入新组件），吞吐量大了再上 Kafka。
