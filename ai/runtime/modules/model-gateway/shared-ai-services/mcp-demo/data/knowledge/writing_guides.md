# AI 写作素材库

## 写作基本原则

好文章的核心在于清晰、简洁、有逻辑。无论是技术文档、商业报告还是创意写作，都应遵循以下原则：

- **明确目标读者**：写之前想清楚谁会读这篇文章，他们的知识水平和期待是什么
- **一个核心观点**：每篇文章只表达一个核心观点，所有段落围绕它展开
- **先说结论**：开头就亮出结论或核心信息，再展开论述（金字塔原理）
- **用简单的词**：能用常见词就不用生僻词，能用短句就不用长句

## 文章结构模板

### 议论文结构

经典的五段论结构适用于大多数议论场景：

1. 引言：提出问题或论点（1 段）
2. 论据一：最强的支撑论据 + 实例（1-2 段）
3. 论据二：第二个支撑论据 + 实例（1-2 段）
4. 反驳预期异议：提前回应可能的反对意见（1 段）
5. 结论：重申论点 + 行动呼吁（1 段）

### 技术文章结构

技术文章建议使用"问题-方案-实现"结构：

1. 问题描述：读者遇到了什么痛点
2. 方案选型：有哪些候选方案，为什么选这个
3. 实现步骤：分步骤的具体操作指南
4. 效果验证：方案实施后的效果数据
5. 总结与展望：经验教训和下一步计划

## 常用修辞技巧

### 类比法

将复杂概念比作读者熟悉的事物。例如：
- "数据库索引就像一本书的目录，帮你快速找到需要的内容"
- "微服务架构就像一个餐厅——前台、厨房、收银各司其职"

### 数据支撑

用具体数字增强说服力：
- 弱：这个方案显著提升了性能
- 强：这个方案将 API 响应时间从 2.3 秒降低到 180 毫秒，提升了 12 倍

### 故事开头

用一个小故事或场景引入主题，比直接说理更吸引人：
- "上周五下午 3 点，我们的监控系统突然报警——CPU 使用率飙升到 98%。这篇文章记录了我们如何在 2 小时内定位并解决这个问题。"

## 标题写作技巧

好标题需要满足三个条件：准确、吸引人、有信息量。

- **数字型**："5 个让你的 Python 代码提速 10 倍的技巧"
- **问题型**："为什么你的 API 总是超时？"
- **方案型**："从零搭建高可用 Redis 集群：完整指南"
- **对比型**："REST vs GraphQL：2024 年该选哪个？"

避免标题党：标题承诺的内容，正文必须兑现。

## AI 辅助写作建议

使用 AI 辅助写作时，注意以下要点：

- **AI 擅长**：生成初稿、扩展要点、调整语气、翻译润色、生成大纲
- **AI 不擅长**：原创观点、深度行业洞察、最新事实核查、个人经历叙述
- **最佳实践**：用 AI 生成初稿框架，人工填充核心观点和专业见解，再用 AI 润色语言
- **必须核查**：AI 生成的数据、引用、技术细节都需要人工验证准确性
