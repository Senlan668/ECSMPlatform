# 记忆服务测试指南

服务提供 6 个 Tool，分两组测试：对话记忆（3 个）和用户画像（3 个）。

---

## 启动服务

```bash
cd mcp-demo
uv run shared/memory_service/server.py
```

看到这行就成功了：

```
Uvicorn running on http://127.0.0.1:9003 (Press CTRL+C to quit)
```

## 测试流程图解

### 对话记忆测试流程

```
test_memory_service.py              记忆服务 (:9003)           SQLite
   │                                   │                        │
   │── save_memory x4 ───────────────→│── INSERT ─────────────→│ memory_messages
   │←─ {status:ok, id:1~4} ──────────│                        │
   │                                   │                        │
   │── recall_memory ────────────────→│── SELECT DESC+reverse →│ 取最新 N 条
   │←─ {messages:[...], count:4} ────│                        │   翻转成正序
   │                                   │                        │
   │── recall (不同 session) ────────→│── SELECT ─────────────→│ 0 条 → 隔离
   │                                   │                        │
   │── clear_memory ─────────────────→│── DELETE ─────────────→│ 清空该会话
   │←─ {deleted:4} ──────────────────│                        │
```

```
test_user_profile.py                记忆服务 (:9003)           SQLite
```

## 常见问题

### Q: 端口被占用？

```bash
# Windows
netstat -ano | findstr "9003"
# Mac/Linux
lsof -i :9003
```

杀掉旧进程后重启即可。

### Q: 想清空所有数据？

```bash
Remove-Item data\sqlite\memory.db    # Windows PowerShell
rm data/sqlite/memory.db             # Mac/Linux
```

重启服务后自动创建新的空数据库。

### Q: recall 返回条数比预期多？

说明有历史残留数据。先跑 `clear_memory` 或删数据库文件后重测。
