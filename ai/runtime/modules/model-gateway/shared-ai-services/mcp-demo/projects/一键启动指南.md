# MCP 多项目共享服务 — 一键启动指南

## 架构总览

```
用户浏览器
  ├── 智能客服 (:8501) ──┐
  └── 写作助手 (:8502) ──┤
                         ▼
                   HTTP 网关 (:8000)
                   ┌────┴────┐
            ┌──────┴──────┐  │
            ▼      ▼      ▼  ▼
        LLM网关  RAG服务  记忆  Prompt中心
        :9001   :9002   :9003  :9004
```

所有业务项目通过 **HTTP 网关** 统一访问 MCP 共享服务，不直接连接 MCP 端口。

---

## 1. 环境要求

| 项目 | 要求 |
|------|------|
| Python | >= 3.11 |
| 包管理 | [uv](https://docs.astral.sh/uv/)（推荐）或 pip |
| 操作系统 | Windows / macOS / Linux |

---

## 2. 安装依赖

```bash
cd mcp-demo
uv sync
```

核心依赖：`mcp[cli]`、`openai`、`httpx`、`fastapi`、`uvicorn`、`chromadb`、`pyyaml`、`python-dotenv`

---

## 3. 配置 `.env`

```bash
cp .env.example .env
```

编辑 `.env`，填入你的火山引擎 ARK 配置：

```env
# 必填：火山引擎 API Key（在控制台 → API Key 管理中获取）
ARK_API_KEY=your-ark-api-key-here

# 通常不需要改
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3

# 必填：在火山引擎控制台创建推理接入点后获得的模型 ID
ARK_CHAT_MODEL=your-chat-endpoint-id
ARK_EMBEDDING_MODEL=doubao-embedding-vision-250615
```

> **注意**：`ARK_API_KEY` 必须是纯英文/数字的真实 Key，不能包含中文。

---

## 4. 一键启动

```bash
cd mcp-demo
uv run scripts/start_all_services.py
```

启动器会按以下顺序自动拉起全部 7 个服务：

| 阶段 | 服务 | 端口 |
|------|------|------|
| ① MCP 共享服务 | LLM 网关 | 9001 |
| | RAG 服务 | 9002 |
| | 记忆服务 | 9003 |
| | Prompt 中心 | 9004 |
| ② HTTP 网关 | FastAPI 治理层 | 8000 |
| ③ 业务项目 | 智能客服 | 8501 |
| | 写作助手 | 8502 |

- 启动前**自动检测端口占用**，若冲突则终止旧进程
- 加 `--no-kill` 参数可跳过已占用端口（不杀进程）
- 每个服务在**独立的 CMD 窗口**中运行

---

## 5. 访问业务项目

启动完成后，浏览器打开：

| 项目 | 地址 | 说明 |
|------|------|------|
| 智能客服 | http://127.0.0.1:8501 | 带知识库检索 + 用户画像的客服对话 |
| 写作助手 | http://127.0.0.1:8502 | 可设置文风的写作对话 |

### 智能客服 API

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/chat` | 对话聊天 |
| POST | `/api/clear` | 清空会话记忆 |
| POST | `/api/profile` | 查看用户画像 |

### 写作助手 API

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/chat` | 写作对话 |
| POST | `/api/style` | 设置文风 |
| POST | `/api/clear` | 清空会话记忆 |
| POST | `/api/profile` | 查看用户画像 |

---

## 6. 停止服务

在启动脚本所在终端按 **Ctrl+C**，会自动优雅终止全部子进程。

---

## 7. 单独启动（调试用）

如需单独启动某个服务进行调试：

```bash
cd mcp-demo

# MCP 共享服务（各启一个终端）
uv run shared/llm_gateway/server.py
uv run shared/rag_service/server.py
uv run shared/memory_service/server.py
uv run shared/prompt_hub/server.py

# HTTP 网关
uv run -m uvicorn gateway.main:app --port 8000

# 业务项目（带热重载）
uv run -m uvicorn projects.customer_service.web:app --port 8501 --reload
uv run -m uvicorn projects.writing_assistant.web_app:app --port 8502 --reload
```

> 启动顺序：MCP 共享服务 → HTTP 网关 → 业务项目

---

## 8. 项目内部调用链路

```
浏览器 → POST /api/chat
       → CustomerServiceAgent / WritingAssistantAgent
         → GatewayClient(api_key) → HTTP 网关 (:8000)
           → MCP Client → memory-service   (recall/save)
           → MCP Client → rag-service      (search)
           → MCP Client → prompt-hub       (get_prompt)
           → MCP Client → llm-gateway      (chat_completion)
         ← JSON 响应
       ← {"reply": "...", "session_id": "..."}
```

## 常见问题

| 问题 | 解决 |
|------|------|
| `NameError: LOG_DIR` | 更新 `start_all_services.py` 到最新版本 |
| `ascii codec can't encode` | 检查 `.env` 中 `ARK_API_KEY` 是否包含中文 |
| 端口被占用 | 默认会自动杀旧进程；或加 `--no-kill` 跳过 |
| 模型响应慢 | `.env` 中 `ARK_CHAT_MODEL` 换成 flash 模型 |
